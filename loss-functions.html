1. What is a Loss Function?

A loss function (also called a cost function or objective function) quantifies how far a model's prediction is from the true value. Given a model \( f(x; \theta) \) with parameters \( \theta \), the loss for a single example is:

$$\mathcal{L}(f(x; \theta), y)$$

Over a dataset of \( N \) examples, we minimize the average loss:

$$\mathcal{L}_{\text{total}} = \frac{1}{N} \sum_{i=1}^{N} \mathcal{L}(f(x_i; \theta), y_i)$$

The training loop computes the gradient of this loss with respect to \( \theta \) and updates parameters to reduce it. The choice of loss function directly determines what the model optimizes and how it behaves.

2. Types of Loss Functions

Loss functions are typically chosen based on the type of task:

TaskTypical Loss Functions
RegressionMSE, MAE, Huber
Binary classificationBinary Cross-Entropy, Hinge
Multi-class classificationCategorical Cross-Entropy

2.1 Mean Squared Error (MSE)

Used for regression. Penalizes larger errors quadratically.

$$\mathcal{L} = \frac{1}{N} \sum_{i=1}^{N} (\hat{y}_i - y_i)^2$$

The squaring means outliers have outsized influence on the loss.

2.2 Mean Absolute Error (MAE)

Also for regression, but more robust to outliers.

$$\mathcal{L} = \frac{1}{N} \sum_{i=1}^{N} |\hat{y}_i - y_i|$$

2.3 Binary Cross-Entropy (BCE)

Used for binary classification. \( y \in \{0, 1\} \), \( \hat{y} \in (0, 1) \).

$$\mathcal{L} = -\frac{1}{N} \sum_{i=1}^{N} \left[ y_i \log(\hat{y}_i) + (1 - y_i) \log(1 - \hat{y}_i) \right]$$

2.4 Categorical Cross-Entropy

Used for multi-class classification. The focus of this document — explained in detail below.

2.5 Hinge Loss

Used in SVMs. Encourages a margin between classes.

$$\mathcal{L} = \max(0, 1 - y \cdot \hat{y})$$

Here \( y \in \{-1, +1\} \).

2.6 Huber Loss

A compromise between MSE and MAE. Behaves like MSE for small errors, MAE for large errors.

$$\mathcal{L} = \begin{cases} \frac{1}{2}(\hat{y} - y)^2 & \text{if } |\hat{y} - y| \leq \delta \\ \delta \, |\hat{y} - y| - \frac{1}{2}\delta^2 & \text{otherwise} \end{cases}$$

3. Information Theory Background

To understand cross-entropy, we need three concepts from information theory:

3.1 Entropy

Entropy measures the uncertainty or information content of a probability distribution \( p \):

$$H(p) = -\sum_{x} p(x) \log p(x)$$

A uniform distribution (all outcomes equally likely) has maximum entropy. A peaked distribution (one outcome almost certain) has near-zero entropy.

3.2 KL Divergence

The Kullback-Leibler divergence measures how much one distribution \( q \) differs from a reference distribution \( p \):

$$D_{\text{KL}}(p \,\|\, q) = \sum_{x} p(x) \log \frac{p(x)}{q(x)}$$

It is always non-negative and equals zero only when \( p = q \). Note: KL divergence is not symmetric.

3.3 Relationship

Cross-entropy decomposes as:

$$H(p, q) = H(p) + D_{\text{KL}}(p \,\|\, q)$$

Since \( H(p) \) is fixed by the true labels, minimizing cross-entropy is equivalent to minimizing KL divergence — i.e., making the predicted distribution match the true distribution.

4. Cross-Entropy Loss: Deep Dive

4.1 Deriving Cross-Entropy from Entropy

The cross-entropy of a true distribution \( p \) and a predicted distribution \( q \) is:

$$H(p, q) = -\sum_{x} p(x) \log q(x)$$

This measures the average number of bits needed to encode events drawn from \( p \) using a code optimal for \( q \). If \( q \) matches \( p \) perfectly, cross-entropy equals entropy. The worse the match, the higher the cross-entropy.

4.2 Cross-Entropy for Classification

In classification, the true distribution is a one-hot vector. For a problem with \( C \) classes and true class \( c \):

$$p = [0, 0, \ldots, 1, \ldots, 0] \quad \text{(1 at position } c \text{, 0 elsewhere)}$$

Plugging into the cross-entropy formula, since \( p(x) = 0 \) for all wrong classes, every term vanishes except the true class:

$$H(p, q) = -\sum_{x} p(x) \log q(x) = -\log q(c)$$

So cross-entropy for a single classification example simplifies to just the negative log-likelihood of the correct class's predicted probability.

4.3 Fusing with Softmax

The model produces raw logits \( \mathbf{z} = [z_1, z_2, \ldots, z_C] \). To get a probability distribution, we apply softmax:

$$q_i = \frac{\exp(z_i)}{\sum_{j} \exp(z_j)}$$

Substituting into the loss:

$$\mathcal{L} = -\log q(c) = -\log \frac{\exp(z_c)}{\sum_{j} \exp(z_j)}$$
$$\mathcal{L} = -z_c + \log \left( \sum_{j} \exp(z_j) \right)$$

This is exactly what nn.CrossEntropyLoss computes in PyTorch — it fuses softmax + negative log-likelihood into a single operation. This is done for numerical stability: computing softmax separately can lead to overflow in the exponentials, while the fused form allows the use of the log-sum-exp trick.

For a batch of \( N \) examples:

$$\mathcal{L} = -\frac{1}{N} \sum_{i=1}^{N} \left[ -z_{i,c_i} + \log \left( \sum_{j} \exp(z_{i,j}) \right) \right]$$

4.4 Intuition

Model predicts\( q(c) \)\( \text{Loss} = -\log q(c) \)Meaning
Correct class with high confidence0.990.01Near-zero penalty
Correct class with moderate confidence0.700.36Small penalty
Correct class with low confidence0.102.30Large penalty
Correct class with near-zero confidence0.014.61Huge penalty
Correct class with zero confidence0.00\( \infty \)Infinite penalty

The \( -\log \) creates an exponentially growing penalty as the model becomes more confidently wrong. This is the key property that makes cross-entropy effective for classification — it strongly discourages the model from assigning near-zero probability to the correct class.

5. Worked Example

3-Class Classification

Suppose we have 3 classes: Cat, Dog, Bird. The true label is Dog (class index 1).

The model outputs raw logits:

$$\mathbf{z} = [2.0, 1.0, 0.1] \quad \text{(Cat=2.0, Dog=1.0, Bird=0.1)}$$

Step 1: Softmax

$$\exp(\mathbf{z}) = [7.389, 2.718, 0.105] \quad \text{sum} = 10.212$$
$$\mathbf{q} = [0.723, 0.266, 0.010]$$

The model assigns 72.3% to Cat, 26.6% to Dog, 1.0% to Bird.

Step 2: Cross-Entropy

True class is Dog (index 1), so we take \( q(1) = 0.266 \):

$$\mathcal{L} = -\log(0.266) = 1.324$$

Step 3: Compare — what if the model had predicted correctly?

If the logits were [1.0, 2.0, 0.1] instead (Dog highest):

$$\mathbf{q} = [0.266, 0.723, 0.010]$$
$$\mathcal{L} = -\log(0.723) = 0.324$$

The loss drops from 1.324 to 0.324 — the model is rewarded for assigning higher probability to the correct class.

Step 4: PyTorch equivalent

import torch
import torch.nn as nn

logits = torch.tensor([[2.0, 1.0, 0.1]])  # shape: (1, 3)
target = torch.tensor([1])                  # Dog = class index 1

loss_fn = nn.CrossEntropyLoss()
loss = loss_fn(logits, target)              # tensor(1.3244)

Cross-entropy loss is the standard choice for classification because:

  • It derives from information theory — minimizing it makes the predicted distribution match the true distribution.
  • The \( -\log \) penalty grows exponentially for confidently wrong predictions.
  • Fused with softmax, it is numerically stable and efficient to compute.
  • Its gradient is simply \( \mathbf{q} - \mathbf{p} \) (softmax output minus one-hot target), which is clean and well-behaved for gradient descent.
